# 小河狸发票助手更新日志

## 1.3.3

- 发票主表新增“入账 / 未入账 / 已入账”筛选，便于快速查看未入账发票。
- 发票台账底部筛选合计新增已入账、未入账数量。
- 发票明细不再展示入账日期、凭证号，避免同一张发票多行明细重复维护造成误解。
- 发票明细导出同步移除入账日期、凭证号字段。
- 分页文案改为“第 N 页 / 共 N 页”，总条数统一放在筛选合计中展示。
- 发票台账表格区域加高，减少页面留白。

## 1.3.2

- 压缩发票台账中的入账日期、凭证号输入框，减少表格横向占用。
- “查看发票”下的 PDF、OFD、XML 入口缩小字号并取消加粗，台账阅读更紧凑。
- 修复分析看板切换公司或日期后部分图表为空的问题，图表实例会重新构建并自动调整尺寸。
- 图表无数据时显示空状态，避免误认为页面卡住。

## 1.3.1

- 新增本机访问保护，可在分享链接页面开启访问密码锁。
- 开启后关闭软件再打开需要输入访问密码，适合个人电脑的轻量保护场景。
- 入账日期改为日期框录入，并增加自动格式整理和行内提示，避免弹窗无法关闭影响修改。
- 本机访问保护不替代分享链接权限，分享链接仍按 token、过期时间和只读/编辑权限控制。

## 1.3.0

- 新增入账日期、凭证号，发票主表可直接维护入账和凭证信息。
- 新增电子发票归档文件夹，支持扫描 PDF、OFD、XML，并按发票号码自动关联到台账。
- 发票台账新增“查看发票”，已归档文件可调用系统默认程序打开，未安装对应软件时给出提示。
- 台账表格支持横向滚动，移除空置的发票代码展示列，避免压缩关键信息。
- 导出台账新增电子发票文件信息，便于核对归档情况。
- 本机 Skill API 增加附件清单和本机打开能力，智能体可识别已归档电子发票。
## 1.2.10

- 增强更新检查可靠性，新增本机后端代理检查更新。
- 前端更新检查增加 GitHub Raw 与 JSDelivr 多地址兜底，减少网络或浏览器策略导致的无提示情况。
- 更新检查失败时在浏览器控制台记录原因，便于后续定位。

## 1.2.9

- 新增本机 Skill API，支持 OpenClaw、WorkBuddy 等智能体读取本机发票台账、明细、汇总和排行数据。
- 发票台账筛选行优化为日历图标选择日期范围，销方/购方合并为“单位名称”检索。
- 单位名称检索按方向智能匹配：开具默认查购方，取得默认查销方，全部方向同时查销方和购方。
- 辅助子表检索同步支持日期、状态、发票号码、数电发票号码和单位名称筛选。
- 修复 AI 本地统计对购销方名称、税号标签、状态条件和“收到/开具给”等自然语言句式的识别问题。

## 1.2.5

- 跨公司导入改为硬校验：开具发票只比较“销方识别号”，取得发票只比较“购方识别号”。
- 公司税号支持填写多个税号，系统会按税号集合比较，任一匹配才允许导入。
- 发票主表和明细表新增原始行 JSON 留存，确保 Excel 原始字段不因台账展示字段精简而丢失。
- 导入结果提示框改为局部高亮，仅突出异常公司、税号和数字，普通统计信息保持正常颜色。

## 1.2.4

- 公司税号识别改为多列名兼容，并规范化比较税号。
- 同一主体存在历史税号和统一社会信用代码时，系统按税号集合判断，避免误拦或漏拦。
- 当前公司已有税号但文件主体税号无法识别时，系统拒绝导入，避免跨公司误入。
- 导入批次时间改为本机时间写入，修复显示时间偏差。
- 发票台账和分析看板日期输入统一为 yyyy-mm-dd，并自动限制格式。
- 优化启动器，减少黑色命令窗口并尝试把浏览器打开到前台。

## 1.2.3

- 修复开具/取得方向选错仍可能导入的问题，安装版接口已验证选反方向会拒绝导入。
- 公司主体税号不一致时拒绝导入，避免把其他公司发票写入当前公司。
- 安装和启动时会识别并关闭旧版后台服务，避免新版安装后页面仍显示旧版本。

## 1.2.1

- 加强导入方向识别：开具发票清单应表现为销售方主体集中，取得发票清单应表现为购买方主体集中。
- 当用户把开具清单选为取得导入，或把取得清单选为开具导入时，系统会拒绝导入且不写入数据。
- 导入其他公司数据时，如果当前公司/历史主体税号与文件主体税号不一致，系统会拒绝导入。
- 如果税号缺失、只能按名称判断主体异常，系统允许继续导入，但会以红色强提醒提示用户确认是否选错公司。

## 1.2.0

- 新增多公司管理，支持新增公司和修改公司名称。
- 数据导入按公司写入，发票唯一键调整为公司内去重和更新。
- 发票台账、辅助子表检索和分析看板支持公司选择。
- 分享链接支持选择单个公司或多个公司，并在令牌列表显示公司名称。
- AI 问答和本地统计按当前公司口径读取数据。
- 升级时自动修复历史数据批次归属，避免出现台账有数据但最近导入为空、无法删除或更新的情况。
- 新版启动时会优先关闭本机旧版服务，避免浏览器仍连接旧端口导致顶栏显示旧版本。
